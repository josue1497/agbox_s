Estructura del framework

app_name/ "nombre de la aplicacion alojada en el servidor".
----Config/ "configuracion de la aplicacion".
--------Framework/ "parseo de url, manejo de peticiones y carga de controladores".
--------core.php "configuraciones de la aplicacion: parametros de conexion a base de datos, carga de archivos base del framework".
--------abx_db_tables.sql "archivo sql de definicion de tablas y campos de la db".
--------abx_db_keys.sql "archivo sql especificacion de claves primarias, unicas y foraneas".
----Controllers/ "carpea donde se crearan los controladores de la aplicacion. por defecto ya estan creados los de autenticacion, manejo de usuarios, y gestion de menu".
----Core/ "conjunto de archivos base del framework".
--------DB.php "se encarga de el manejo de la base de datos".
--------Model.php "plantilla de tabla de base de datos, implementa los metodos crud de DB".
--------Column.php "implementa atributos y metodos para el mapeo y manejo de columnas de db en un modelo".
--------Controller.php "manejo de acciones de la aplicacion, instancia modelos, carga vistas, gestiona el menu."
--------Component.php "construye elementos para la vista a partir de la definicion del modelo y las columnas".
--------Session.php "manejo de sesiones".
--------View.php "maneja la construccion de vistas".  
----Models/ "se crearan los modelos de las tablas de la db, especificando atributos para la construccion dinamica de elemntos de vistas"
----Views/ "se crean carpetas para las vistas de cada modelo, dentro de ellas, se crean los archivos de cada vista. No son necesarias si en el controlador se especifica que la vista se construira automaticamente. en esta carpeta esta Layout que maneja plantillas html, scripts, estilos, imagenes y vendors".
----Webroot/ "esta carpeta contiene el index que ejecuta el codigo de la aplicacion" 


Instalacion
Crea la base de datos en tu manejador, importa el archivo app_name/Config/apx_db_tables.sql y luego app_name/Config/apx_db_tables.sql. 
Actualiza la informacion del archivo app_name/Config/core.php, especificando nombre de la aplicacion, nombre de la carpeta, y parametros de conexion a la base de datos.

Construir una funcionalidad o modulo
1. Crear la tabla n la base de datos segun la necesidad,
2. Crear el modelo de la tabla, especificando cuales son las columnas, cual es primary_key y cual es name_key (sin esto no se podra persistir nada), y se especifican las relaciones. Tomese como ejemplo el modelo Usuario.  
3. Crear el controlador del modulo, "nombre_controladorController", definir los metodos basicos, como ejemplo el controlador de usuarioController, y definir los metodos adicinales que hagan falta, un metodo para cada vista. en los metodos basicos se especifica si la vista se construye automaticamente o no.
4. si no se desea construir la vista automaticamente, se puede construir de forma manual, en ese caso, se crea una carpeta con el nombre del model, y dentro de ella, un archivo para cada vista. los archivos basicos serian index.php para grid, y form.php para crear y editar. se pueden crear vistas cistomizadas, segun la necesidad, para ejemplo la vista Index/login.php.
